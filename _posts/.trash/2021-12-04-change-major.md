去年今日，我提交了信息学院的转专业申请，奈何成绩不够优秀以失败告终。过去这一年里，我一直没有放弃过自己想要成为App独立开发者的梦想。

# Xcode
高中毕业的时候，我在再三思量之后选择了Mac作为我步入大学的“装备”，放弃了期盼已久的游戏本，主要原因之一就是Xcode。虽然它并没有Visual Studio一样万能，JetBrains全家桶一样易用，但是它一定是你步入App开发大门一个不错的IDE。不仅有能够方便快速实时调试的Playgrounds工具，还有具备许多现代特性的UI设计框架SwiftUI。


# SwiftUI
去年我做过一个关于校园码的快速展示App。由于我当时对响应式设计理解的不够到位，导致最后始终无法让屏幕上面的当前时间做到实时刷新。在过去这一年里，我在学习其它许多框架的同时，重新阅读了一遍SwiftUI开发文档。终于找到了定时器`Timer()`*使用方法，并成功做到了实时刷新时间。
![校园码示例](/image/qrcode.png)

> *注解：通过一个放在主线程上的计时器、以一秒钟的刷新周期初始化之后让他被显示时间的视图View中添加的.onReceive()捕获（相当于让创建好的定时器每秒钟发送的消息传给onReceive()方法，然后onRecieve会响应消息，执行一个闭包，此时在闭包中写入Date()类的构造函数，即可达到获取当前系统时间的目的
>> 但是这种获取时间的方式存在着内存泄漏问题（有可能），对染能变已通过并使用，但是在Canvas中debug时仍然会使电脑占用过多内存（一般在16GB以上）


# 越狱iOS插件设计&App逆向工程
我还接触到了越狱后的iOS系统的Cydia Substrate插件开发，在一个手心大小的iPhone SE上使用apt配置环境、安装依赖包。学会了使用ssh和简单的vim使用方法。    
在App Store中分发的.ipa安装包经过加密，使得Mach-O文件变得无法分析。配合越狱iPhone，便能够做到通过clutch工具在终端中将安装在iPhone上的App重新打包成解密版本。    
我的方法是在SSH中通过clutch解密App之后再使用scp命令将文件下载到本地

# 开发文档的阅读
- 一路学习过来，没有一位老师。只有志同道合的伙伴曾经帮助过我解决过一些问题。剩下的全部靠自己去学习，去哪里学习呢？得力于自己的英语能力（虽然不是非常好，但是阅读纯英文的开发文档还是能够胜任的），我会去查阅官网发布的英文文档。我比较不相信网上许多速成的视频课程，因为我一直认为掌握某项技能是一个系统性的学习过程，掌握核心的功能确实很快，但是要做到信手拈来的能力，即创造力能跟上想象力的境界还是需要完整理解开发文档才可以。
![Read Documentation](/image/xcode-documentation.png)

- 一般我阅读开发文档的步骤大致为：
1. 通读一遍，同时搭配上实机操作
2. 按照给出的章节顺序整理出自己平常工作流中时常会用到的功能
3. 对上述重要功能进行重点巩固


# Github
作为一位新手开发者，不可避免的会在平常造一些轮子。并且在自己写的内容上进行版本控制的意识比较淡薄。在Github上，你不仅可以学习版本控制工具git的操作，还能够找到自己感兴趣的开源仓库并优雅的阅读最新版本的源代码。对于中后期学习编程有不小的帮助。 
我除了在GitHub上传过自己制作的E浙理客户端的重构版本之外，还开设了这个静态网站GitHub-Page。这个网站平常作为自己记录编程学习心得的博客一直在更新。经过几次尝试之后，我觉得写博客的确是一个巩固与总结已经学习的知识的好地方，降低了整理笔记过程的枯燥——因为你能够与他人分享你的笔记并有可能会帮助到其他人，很有收获感！


# LeetCode
关于算法、数据结构和动态规划，我知道自己学习的程度还远远不够。因此，空闲时间我也会上力扣平台刷一些题目。虽然到目前为止，通过率勉强能过25%，但是我相信如果我能来到信息学院，一定会在对应专业课上努力把他们都牢牢掌握。   
因为我认为未来编程定会迎来低代码平台的变革，到那个时候，程序员与普通人的区别也许就在于程序员掌握了“机器的语言”，许多人类的逻辑放在计算机上就需要用严谨的算法去描述。对于一些业务逻辑的性能优化，只有程序员才能够深入问题核心。我们常将专业素养不高的程序员写出来的代码称作**“屎山”**为何？一个if语句具有上百个分支，debug到新情况就加else if捕获。这样的代码能不烂吗

# WWDC
很遗憾，大一的时候我没有加入到学校的acm队伍中，可能当时我虽然有雄心壮志但是实力还不够支持自己去参加吧。不过，我的一个期盼已久的梦想便是能在WWDC学生挑战赛中获奖。去年我便去浙江大学参加过一次有关WWDC的讲座活动，线上的直播更是参加过更多次。
